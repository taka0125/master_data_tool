# Worktree DB Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 各 git worktree が独立した MySQL コンテナを持ち、bundle volume を共有できるようにする。

**Architecture:** `compose.yaml` から `container_name` を削除し bundle volume を external 化。`scripts/worktree-setup.sh` が `COMPOSE_PROJECT_NAME` を導出してコンテナ起動・DB 作成・マイグレーション・`.envrc` 生成を一括実行する。

**Tech Stack:** Docker Compose v2, bash, MySQL 8, Ruby (ridgepole for migrations)

### Task 1: compose.yaml を修正

**Files:**
- Modify: `compose.yaml`

- [ ] **Step 1: `container_name` 行を削除、`bundle` volume を external 化**

`compose.yaml` を以下の内容に置き換える：

```yaml
services:
  mysql:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
    ports:
      - '127.0.0.1::3306'
    volumes:
      - mysql:/var/lib/mysql
    command: >
      mysqld 
        --sql-mode=NO_ENGINE_SUBSTITUTION
        --character-set-server=utf8mb4
        --innodb-file-per-table=1

  ruby:
    image: ruby:3.2
    command: sleep infinity
    working_dir: /app
    volumes:
      - .:/app:cached
      - bundle:/bundle
    environment:
      - HISTFILE=/app/.bash_history
      - BUNDLE_PATH=/bundle
      - BUNDLE_JOBS=4
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_USER
      - DB_PASSWORD
      - DB_NAME
    depends_on:
      - mysql

volumes:
  mysql:
  bundle:
    external: true
    name: master_data_tool_bundle
```

変更点は2箇所のみ：
1. `container_name: master_data_tool-mysql` を削除（mysql サービスから）
2. `bundle:` の定義を `external: true` + `name: master_data_tool_bundle` に変更

### Task 2: scripts/worktree-setup.sh を作成

**Files:**
- Create: `scripts/worktree-setup.sh`

- [ ] **Step 1: スクリプト作成**

```bash
#!/bin/bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "${SCRIPT_DIR}/.." && pwd)"

cd "${REPO_ROOT}"

# Load .env.test
set -a
# shellcheck source=../.env.test
source "${REPO_ROOT}/.env.test"
set +a

# Derive COMPOSE_PROJECT_NAME from current directory name
PROJECT_NAME=$(basename "${REPO_ROOT}" | tr '[:upper:]' '[:lower:]' | sed 's/[^a-z0-9_-]/-/g')
export COMPOSE_PROJECT_NAME="${PROJECT_NAME}"

DOCKER_COMPOSE="docker compose --env-file ${REPO_ROOT}/.env.test"

echo "==> COMPOSE_PROJECT_NAME: ${COMPOSE_PROJECT_NAME}"

# Step 1: Create shared bundle volume (idempotent)
if ! docker volume inspect master_data_tool_bundle >/dev/null 2>&1; then
  echo "==> Creating shared bundle volume: master_data_tool_bundle"
  docker volume create master_data_tool_bundle
else
  echo "==> Bundle volume already exists: master_data_tool_bundle"
fi

# Step 2: Start containers
echo "==> Starting containers..."
${DOCKER_COMPOSE} up -d

# Step 3: Wait for MySQL (max 30 seconds)
echo "==> Waiting for MySQL..."
MAX_WAIT=30
count=0
until ${DOCKER_COMPOSE} exec -T mysql mysql -u "${DB_USER}" -p"${DB_PASSWORD}" -e "SELECT 1" >/dev/null 2>&1; do
  count=$((count + 1))
  if [ "${count}" -ge "${MAX_WAIT}" ]; then
    echo "Error: MySQL did not become ready within ${MAX_WAIT} seconds"
    exit 1
  fi
  echo "  waiting... (${count}/${MAX_WAIT})"
  sleep 1
done
echo "==> MySQL is ready"

# Step 4: Create database (idempotent)
echo "==> Creating database: ${DB_NAME}"
${DOCKER_COMPOSE} exec -T mysql \
  mysql -u "${DB_USER}" -p"${DB_PASSWORD}" \
  -e "CREATE DATABASE IF NOT EXISTS \`${DB_NAME}\`"

# Step 5: Run migrations
echo "==> Running migrations..."
${DOCKER_COMPOSE} exec -T ruby bash -c './scripts/migrate.sh'

# Step 6: Get actual host port and generate .envrc
HOST_PORT=$(${DOCKER_COMPOSE} port mysql 3306 | cut -d: -f2)
echo "==> MySQL host port: ${HOST_PORT}"

cat > "${REPO_ROOT}/.envrc" <<EOF
# auto-generated by scripts/worktree-setup.sh
# DO NOT EDIT MANUALLY
export COMPOSE_PROJECT_NAME=${COMPOSE_PROJECT_NAME}
export DB_HOST=127.0.0.1
export DB_PORT=${HOST_PORT}
EOF

echo ""
echo "==> Setup complete!"
echo ""
echo "Run one of the following to activate environment variables:"
echo "  direnv allow"
echo "  source .envrc"
```
