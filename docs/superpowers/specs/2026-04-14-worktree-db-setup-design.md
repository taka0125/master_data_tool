# Worktree DB Setup Design

Date: 2026-04-14

## 概要

superpowers の git worktree ベース開発において、各 worktree が独立した DB コンテナを持てるようにするためのスクリプトと設定変更の設計。

## 背景・問題

- `compose.yaml` の `container_name: master_data_tool-mysql` がハードコードされており、複数 worktree を同時起動すると Docker コンテナ名が衝突する
- worktree は `worktrees/<branch>` 以下に作成される（リポジトリ内サブディレクトリ）
- bundle (gem キャッシュ) は worktree 間で共有したい
- MySQL データは worktree ごとに独立させたい
- `.env.test` は GitHub Actions で使用しているため廃止不可

## 設計

### compose.yaml の変更

2点のみ変更：

1. `container_name: master_data_tool-mysql` を削除
   - Docker Compose が `{COMPOSE_PROJECT_NAME}-mysql-1` で自動命名するため、worktree ごとにコンテナ名が分離される

2. `bundle` volume を `external: true` に変更し、固定名 `master_data_tool_bundle` を指定
   - 全 worktree で同一の bundle volume を共有
   - `mysql` volume はプロジェクト名に追従するため、worktree ごとに自動分離される

```yaml
volumes:
  mysql:
  bundle:
    external: true
    name: master_data_tool_bundle
```

### scripts/worktree-setup.sh

各 worktree 作成後に実行するセットアップスクリプト。

**処理フロー:**

1. `COMPOSE_PROJECT_NAME` を決定（カレントディレクトリ名を英数字・ハイフンに正規化）
2. `master_data_tool_bundle` volume の冪等作成（存在確認 → なければ `docker volume create`）
3. `COMPOSE_PROJECT_NAME` を export して `docker compose up -d`
4. MySQL 起動待機（最大30秒ループ、`docker compose exec` で接続確認）
5. DB 作成（`docker compose exec mysql` で直接 SQL 実行 — `make mysql/create_db` ではなくスクリプト内で完結）
6. マイグレーション（`docker compose exec ruby bash -c './scripts/migrate.sh'` で実行）
7. `.envrc` 生成（`docker compose port mysql 3306` で実際のホストポートを取得）
8. 完了メッセージ（`direnv allow` 案内）

### 生成される .envrc

```bash
# auto-generated by scripts/worktree-setup.sh
export COMPOSE_PROJECT_NAME=<ディレクトリ名>
export DB_HOST=127.0.0.1
export DB_PORT=<docker compose port で取得した実際のホストポート>
```

`.envrc` は `.gitignore` 管理のためコミットされない。

### Makefile

変更なし。`DOCKER_COMPOSE = docker compose --env-file .env.test` を維持。

Docker Compose は環境変数の `COMPOSE_PROJECT_NAME` を `--env-file` より優先して参照するため、`.envrc` を `source` した状態で `make` を実行すれば自動的に正しい worktree のプロジェクトに向く。

## 役割分担

| 項目 | CI（GitHub Actions） | ローカル worktree |
|------|---------------------|-----------------|
| `.env.test` | 使用（DB_PORT=3306固定） | 使用（`--env-file`経由） |
| `COMPOSE_PROJECT_NAME` | なし（ディレクトリ名デフォルト） | `.envrc` で設定 |
| コンテナ内 DB 接続 | `DB_HOST=mysql, PORT=3306` | 同左（compose.yaml の env） |
| ホスト側 DB 接続 | 不要 | `.envrc` の `DB_PORT` を使用 |

## 利用フロー

```bash
# worktree 作成（superpowers が行う）
git worktree add worktrees/feature-foo feature/foo

# DB セットアップ
cd worktrees/feature-foo
bash scripts/worktree-setup.sh

# direnv 有効化
direnv allow   # または source .envrc

# テスト実行
make ruby/rspec
```

## 初回セットアップ（メインリポジトリ）

`worktree-setup.sh` はメインリポジトリでも実行可能な冪等設計にする。既存の `make setup` の代替として機能する。

```bash
# メインリポジトリ初回セットアップ
bash scripts/worktree-setup.sh
direnv allow
```

`docker volume create master_data_tool_bundle` はスクリプト内で冪等に処理するため、手動実行不要。

## ファイル変更サマリ

| ファイル | 変更内容 |
|---------|---------|
| `compose.yaml` | `container_name` 削除、`bundle` volume を external 化 |
| `scripts/worktree-setup.sh` | 新規作成 |
| `Makefile` | `worktree/setup` ターゲット追加 |
| `.envrc` | スクリプトが自動生成（git 管理外） |
